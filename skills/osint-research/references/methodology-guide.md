# 方法论操作指南

## 螺旋迭代模型详解

### 模型概述

螺旋迭代是一种渐进式的信息收集方法，通过多轮迭代逐步扩展信息范围，每一轮都基于上一轮的结果发现新的线索。

```
初始种子 → 信息收集 → 分析验证 → 发现新线索 → 扩大搜索 → 循环迭代
     ↑___________________________________________________________|
```

### 迭代流程

#### 第1轮：种子启动
- **输入**: 用户提供的初始种子（公司名、人名、产品名）
- **目标**: 收集种子实体的基础信息
- **输出**: 基础信息 + 第一层关联实体

#### 第2轮：关联扩展
- **输入**: 第1轮发现的关联实体
- **目标**: 收集关联实体的详细信息
- **输出**: 扩展信息 + 第二层关联实体

#### 第3轮：深度挖掘
- **输入**: 第2轮发现的新实体
- **目标**: 挖掘更深层的关联关系
- **输出**: 完整的关系网络

### 迭代控制参数

- **最大迭代深度**: 建议2-3轮，避免无限扩散
- **每轮种子数量**: 建议不超过10个，控制数据量
- **停止条件**:
  - 达到最大迭代深度
  - 没有发现新的关联实体
  - 数据量达到预设上限

## 种子扩散策略

### 企业种子扩散

```
目标企业
    ├── 股东 → 投资方企业
    ├── 子公司 → 关联企业
    ├── 投资企业 → 被投企业
    ├── 供应商 → 上游企业
    └── 客户 → 下游企业
```

### 人员种子扩散

```
目标人员
    ├── 任职企业 → 企业网络
    ├── 投资企业 → 投资版图
    ├── 关联人员 → 人脉网络
    └── 历史任职 → 职业轨迹
```

### 产品种子扩散

```
目标产品
    ├── 竞品 → 竞争对手
    ├── 供应商 → 供应链
    ├── 客户 → 市场分布
    └── 技术路线 → 专利布局
```

## 交叉验证方法

### 多源验证
- 同一信息从多个数据源采集
- 对比不同来源的数据一致性
- 标记冲突信息供人工确认

### 时间验证
- 检查信息的时效性
- 对比历史数据变化
- 识别最新状态

### 逻辑验证
- 检查数据的逻辑一致性
- 验证关系链的合理性
- 识别异常数据

## 质量控制

### 数据质量指标
- **覆盖率**: 目标实体的信息完整度
- **准确率**: 信息的一致性程度
- **时效性**: 数据的更新程度

### 质量提升策略
- 多源交叉验证
- 人工抽样检查
- 持续更新维护
