# 投资分析框架指南

## 一、AI-快速分析模式

### 问题分类体系

| 类型 | 关键词 | 分析重点 |
|-----|-------|---------|
| 宏观 | 经济、政策、货币、全球趋势 | 利率、汇率、通胀、政策方向 |
| 行业 | 行业周期、竞争格局、政策支持 | 供需、竞争、技术、监管 |
| 个股 | 公司基本面、财务、估值、竞争 | 业务、财务、估值、治理 |
| 事件 | 地缘政治、政策落地、突发新闻 | 影响路径、受益/受损方 |
| 财务 | 财报、盈利预测、现金流、估值 | 财务指标、预测、估值 |
| 商品 | 供需格局、价格趋势、库存 | 供需、库存、价格、周期 |

### 输出结构模板

```
核心结论：[一句话投资立场]

背景与现状：[大环境 + 当前对象情况]

正向驱动因素：[利好逻辑1-3条]

负向抑制因素：[风险逻辑1-3条]

市场表现与资金面：[成交量、资金流向、情绪指标]

估值与对比：[估值指标 + 同业/历史对比]

机会与催化剂：[可能触发上行的因素]

风险与不确定性：[下行风险点]

操作建议：[仓位建议 + 价格区间 + 时间周期]
```

### 输出风格
- 简洁、直奔要点
- 每部分1-2句话
- 避免冗长背景描述

## 二、AI-深度思考模式

### 思考逻辑（参考DeepSeek R1）

1. **问题解构**：将复杂问题拆解为可分析的子问题
2. **多维度验证**：从不同角度验证假设
3. **反事实思考**：考虑相反情景的可能性
4. **证据权重**：对不同证据赋予不同置信度
5. **置信度表达**：明确说明结论的确定性程度

### 输出结构

```
【分析概览】
- 研究对象
- 分析时间范围
- 核心假设

【深度分析】
1. 第一层分析：[最直接的因素]
2. 第二层分析：[传导机制]
3. 第三层分析：[系统性影响]

【情景分析】
- 乐观情景：[条件 + 结果]
- 基准情景：[条件 + 结果]
- 悲观情景：[条件 + 结果]

【关键变量】
- 上行催化剂：[触发条件]
- 下行风险点：[触发条件]

【置信度评估】
- 核心结论置信度：[高/中/低]
- 关键假设风险点

【投资建议】
- 投资逻辑
- 目标价位
- 止损价位
- 时间框架
```

## 三、华尔街投研四角色模式

### 3.1 研究员（Research Analyst）

#### 宏观/政策分析维度
- 流动性与利率：真实利率、期限利差、信用脉冲
- 通胀与增长：商品/服务/工资拆分、产出缺口
- 汇率与跨境：美元周期、资本流动
- 政策函数：货币/财政/产业/监管方向
- 事件传导：冲击通道矩阵

#### 行业分析维度
- 结构（SCP）：CR3/CR5、上下游议价、进入壁垒
- 供需与利润池：成本曲线、库存周期、价格机制
- 技术曲线：关键节点技术、替代路径
- 监管与标准：准入/补贴/集采/环保
- KPI字典：行业特定领先指标

#### 公司分析维度
- 业务拆解：主营业务、收入/利润结构
- 商业模式：护城河分析（网络效应/规模经济/品牌/渠道）
- 治理与激励：股权结构、管理层激励
- 财务分析：营收确认、毛利率驱动、现金转换
- 估值：多模型估值、同业对比

### 3.2 基金经理（Portfolio Manager）

#### 决策框架
- 投资逻辑审视：是否符合策略？
- 风险收益比评估：预期收益 vs 最大风险
- 组合匹配：相关性、分散度、因子暴露
- 资金分配：仓位大小（核心/卫星/观望）
- 时间框架：短期交易 vs 中长期投资

#### 输出要素
- Regime判定：风险偏好、Beta目标、风格倾向
- 预算分配：风险桶划分
- 对冲工程：对冲工具选择
- 情景集：Bull/Base/Bear概率与收益

### 3.3 交易员（Trader）

#### 执行框架
- 流动性分析：盘口深度、成交量、波动率
- 执行方式：限价/市价/TWAP/VWAP
- 战术设计：建仓节奏、择时信号
- 成本控制：交易成本、冲击成本、滑点

#### 输出要素
- 事件日历与流动性评分
- 执行算法选择
- 期权微结构（如适用）
- TCA基准

### 3.4 风控（Risk Manager）

#### 风险框架
- 单一标的风险：波动率、VaR、最大回撤
- 组合风险：集中度、因子暴露、杠杆
- 合规检查：监管红线、投资范围
- 压力测试：极端情景损益

#### 输出要素
- 敞口与集中度指标
- 流动性与拥挤度指标
- 压力与尾部风险
- 处置曲线与限位规则

### 3.5 四角色整合报告结构

```
【摘要】
- 投资标的与结论
- 推荐仓位与预期收益
- 风险提示

【宏观与行业分析】
- 宏观要点
- 行业周期与政策

【公司基本面】（研究员输出）
- 业务拆解与财务建模
- 竞争格局
- 管理层与治理

【投资逻辑与估值】（基金经理输出）
- 投资亮点与催化剂
- 估值比较
- 投资逻辑总结

【交易执行方案】（交易员输出）
- 买入区间、目标价、止损位
- 执行策略

【风险控制】（风控输出）
- 单标的风险指标
- 组合影响
- 风险对冲建议

【投资结论】
```

## 四、数据质量规范

### 数值精度要求
| 数据类型 | 精度要求 |
|---------|---------|
| 价格数据 | 保留2位小数 |
| 财务比率 | 保留2位小数（百分比） |
| 估值倍数 | 保留1位小数 |
| 收益率 | 保留2位小数（百分比） |
| 风险指标 | 保留2位小数 |

### 数据来源优先级
1. 官方财报（10-K/10-Q/年报/季报）
2. 交易所数据
3. 权威金融数据终端
4. 第三方数据提供商

### 数据验证规则
- 同比/环比变化异常时标注
- 缺失数据必须说明
- 估算数据必须标注"估算"
